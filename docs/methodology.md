# Methodology

This project organizes evidence about IOOS-related economic impacts into a structured matrix. Each row represents one evidence item, use case, metric, or claim that can be traced to a registered source.

The matrix separates impact claims from source metadata:

- `data/evidence_matrix.csv` contains the evidence rows and claim language.
- `data/source_registry.csv` contains source names, URLs, source types, verification status, and notes.
- `data/review_needed.csv` is generated by validation and lists rows or sources that need follow-up.

The workflow is intentionally conservative. Quantified values, modeled estimates, and causal statements are flagged when they need source verification or when attribution to IOOS is not strong enough to support stronger claim language.

## Quarterly Update Workflow

1. Add or update source records in `data/source_registry.csv`.
2. Add evidence rows through the Streamlit dashboard or by editing `data/evidence_matrix.csv`.
3. Run `python scripts/validate_matrix.py`.
4. Review `data/review_needed.csv`.
5. Resolve validation issues or keep limitations visible for reporting.
6. Export summary tables to `outputs/summary_tables/` as needed.
